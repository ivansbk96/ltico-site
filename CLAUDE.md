- Отдельные HTML-страницы + один общий `styles.css`, подключённый через `<link>`.
- Логотип — `logo.png` (реальный логотип компании, три скошенные полосы + «LTI»).
- Никаких localStorage/sessionStorage, никакого внешнего JS. Только ваниль.

Страницы:
- `index.html` — главная (хиро, услуги, шаги работы, о компании, CTA)
- `gabaritnye-gruzy.html` — образцовая страница услуги (хиро со схемой груза,
  таблица параметров, примеры перевозок, FAQ). ШАБЛОН для остальных услуг.
- `kontakty.html` — контакты + форма заявки
- `glossary.html` — глоссарий терминов
- `dokumenty-i-dopuski.html` — документы и допуски по типам грузов
- `robots.txt`, `.htaccess` — служебные файлы

## ГЛАВНОЕ ПРАВИЛО: не выдумывать факты

Это коммерческий сайт реальной компании. НЕЛЬЗЯ придумывать:
- грузоподъёмность, габаритные пороги, тоннаж (в хедлайне сейчас «1,5–20 т» —
  это ориентир с прежнего сайта, не менять без данных от владельца);
- примеры перевозок, кейсы, отзывы клиентов;
- сроки доставки по маршрутам;
- номера законов, точные перечни документов;
- фото техники (нет реальных — не подставлять стоковые).

Там, где данных нет — оставлять видимый плейсхолдер (класс `.note` в вёрстке или
пометка «Сюда впишем …»), а не заполнять правдоподобной выдумкой. Если правка
требует факта, которого нет в файлах, — спросить у владельца, а не сочинить.

## Форма заявки

В `kontakty.html` форма настроена на сервис Web3Forms, но ключ ещё заглушка:
`YOUR_ACCESS_KEY_HERE`. Пока не заменён на реальный ключ с web3forms.com — форма
не отправляет заявки. Не удалять заглушку, не имитировать работающую отправку.

## Публикация (важно понимать про окружение)

Сайт лежит на хостинге Beget. Claude Code правит ТОЛЬКО локальные файлы в этой
папке — он НЕ заливает изменения на сервер. После правок владелец сам загружает
обновлённые файлы на Beget (через файловый менеджер или scp). После значимых
изменений — напоминать про необходимость залить файлы на сервер.

Тестовая версия сейчас публикуется на поддомен `new.ltico.ru` (основной `ltico.ru`
пока со старым сайтом). При публикации styles.css и logo.png должны лежать рядом
с HTML-файлами в одной папке (public_html).

## Стиль работы

- Сохранять единообразие: новые страницы копируют структуру шапки/футера/CTA
  из существующих, чтобы навигация и подвал совпадали на всех страницах.
- При добавлении страницы в навигацию/футер — обновлять ссылки на ВСЕХ страницах.
- Не усложнять: никакой сборки, минимум JS, доступность и мобильная вёрстка важны
  (в styles.css уже есть медиазапросы и поддержка prefers-reduced-motion).
